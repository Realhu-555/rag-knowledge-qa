# AI应用工程师学习路线图

> 来源：https://blog.51cto.com/u_39037/14584878

## 定位

"AI应用工程师"本质上是AI落地的"最后一公里"，负责把实验室里的模型变成用户可用的产品。核心能力：在强大基座模型基础上，不依赖顶级算法团队，能独立或小团队闭环交付AI应用。

---

## 第一阶段：基础工程能力（1-2个月）

**目标**：能独立开发Web服务，能接手常规后端需求。

### 核心语言 Python
- FastAPI/Flask框架，RESTful API
- 异步编程：async/await（处理高并发AI请求的关键）
- 环境管理：Conda/Poetry

### 数据库与缓存
- MySQL：CRUD、索引优化
- Redis：字符串、列表、哈希等数据结构，缓存和简单消息队列
- 向量数据库：Milvus或Qdrant的原理和基础用法（RAG基础）

### 部署与容器化
- Docker：Dockerfile + Docker Compose
- Linux基础：常用命令、文件权限、进程查看

### AI辅助编程
- GitHub Copilot / Cursor

---

## 第二阶段：AI应用核心开发能力（2-3个月）

**目标**：掌握RAG和Agent，能动手做出智能应用。

### 大模型基础认知
- Transformer原理入门（自注意力机制、Token、上下文窗口）
- Prompt Engineering：结构化提示词、少样本学习、思维链

### RAG（检索增强生成）— 核心技能
完整流程：文档加载 → 文本分割 → 向量化（Embedding）→ 检索 → 召回后排序 → 生成答案
- 重点：优化检索质量（业务效果的瓶颈）

### Agent开发
推荐框架：LangGraph（复杂Agent）或 CrewAI（上手快）
- 任务规划：大任务拆解成小步骤
- 工具调用：调用API、查数据库、搜网页
- 记忆管理：短时记忆（对话上下文）+ 长时记忆（用户画像）

### 微调入门（加分项）
- LoRA等轻量级微调技术

---

## 第三阶段：工程落地与系统迭代（持续进行）

**目标**：让AI系统稳定、可靠，达到生产级标准。

### 构建观测与评估体系
- 全链路追踪：LangSmith等工具记录调用输入/输出/耗时/Token消耗
- 效果评测：自动化AI效果评估机制

### 性能优化
- 异步IO处理大并发
- 降级策略：模型超时或报错时的兜底回答

### 沉淀可复用资产
- 封装工具、提示词模板、RAG流程为SDK或内部组件
